<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI input -> orchestrator | URL string from argparse positional argument enters pipeline |
| orchestrator -> crawler.fetch_url() | URL string passed to HTTP fetch; SSRF already prevented in crawler |
| orchestrator -> access_fetcher.fetch_access_signals() | URL string passed; httpx fetch independently |
| Module exceptions -> orchestrator error list | Exception messages captured in errors[] list, displayed to user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | run_pipeline() url param | mitigate | URL originates from argparse (not shell interpolation); no os.system() or subprocess(shell=True) used anywhere in pipeline; crawler.py is_ssrf_safe() blocks internal addresses |
| T-06-02 | Information Disclosure | orchestrator error messages | mitigate | Exception messages are captured via `str(e)` only; no stack traces propagated; error messages from controlled error_type enums (crawler, access fetcher) not raw system errors; verbose flag only prints stage names, never raw HTML or response bodies |
| T-06-03 | Denial of Service | run_pipeline() timeout passthrough | mitigate | `--timeout` arg validated by argparse as float; passed to fetch_url() which has 10s default; crawler.py enforces MAX_RESPONSE_SIZE (10MB) on response bodies |
| T-06-04 | Elevation of Privilege | Module import chain | accept | All imported modules are within project's own src/checker/ namespace; no privilege escalation surface at the composition layer |
</threat_model>
